## BUG-004: GET /catalogApi/getProduct возвращает избыточный объём данных при запросе карточки одного товара

**Severity:** Low
**Priority:** Medium
**Тип:** Performance
**Статус:** Open

---

### Предусловия
1. Товар со slug `gumshoes-vans-7502616` существует
2. X-Locale: ru-kz задан в заголовке запроса
3. X-Site: kz задан в заголовке запроса

### Тестовые данные
- **Endpoint:** `https://front-api.intertop.kz/catalogApi/getProduct`
- **Method:** GET
- **Params:** `body: [{"id":"gumshoes-vans-7502616"}]`

### Шаги воспроизведения
1. Открыть Postman
2. Создать GET-запрос по указанному endpoint
3. Задать указанные параметры и заголовки запроса
4. Нажать Send

### Ожидаемый результат
Ответ возвращает статус код 200 OK и поля `id`, `sku`, `exists: true` и `options`

### Фактический результат
200 OK, все ожидаемые поля присутствуют. Ответ включает данные о наличии товара
всех размеров, во всех магазинах всех городов, включая данные о самих магазинах.

| Метрика | Значение |
|---------|----------|
| Размер ответа | 8.54–8.55 KB (среднее **8.548 KB** по 5 замерам) |
| Количество строк JSON | **4575** |
| TTFB | 123.54–183.35 ms (среднее **152.44 ms** по 5 замерам) |

### Влияние
1. Избыточная нагрузка на сервер при каждом запросе карточки товара
2. Увеличенное время ответа из-за объёма работы на бэкенде
3. Лишнее потребление трафика мобильного интернета

### Окружение
- **OS:** macOS Tahoe 26.4
- **Инструмент:** Postman
