# BUG-013: Фильтр товаров по ценовому диапазону — фильтр не применяется при равных нижней и верхней границах диапазона

| Поле | Значение |
|---|---|
| **ID** | BUG-013 |
| **Серьезность** | Средняя |
| **Приоритет** | Средний |

## Предусловия

1. [Страница товаров](https://intertop.kz/ru-kz/deals/17956/) открыта
2. Пользователь может быть авторизован или нет

## Тестовые данные

1. Диапазон: минимальная цена — минимальная цена (актуальная минимальная цена на момент написания ТС — 12790 ₸, может меняться. Минимальная цена указана в выпадающем списке фильтра цен в левом поле меню)

## Шаги воспроизведения

1. Нажать на кнопку "Цена"
2. В выпадающем меню ввести минимальную цену в поле максимальной цены (правое поле меню), указанную в поле минимальной
3. Нажать на кнопку "Ок"

## Ожидаемый результат

Применяется ценовой фильтр с минимальным и максимальным значениями диапазона, равными минимально допустимой цене. Применённый фильтр отображается под списком фильтров. Показаны все и только товары с ценой равной минимально допустимой цене.

## Фактический результат

Фильтр не применился и не отобразился под списком фильтров, содержимое страницы не обновилось и не включает в себя все и только товары в ожидаемом ценовом диапазоне.

При попытке применить фильтр с равными значениями диапазона (12790-12790) запрос `getDealsFilters` во вкладке Network в DevTools не зафиксирован — запрос на сервер не отправляется.

При отправке этого же запроса в Postman с тем же диапазоном сервер возвращает список товаров, идентичный списку при диапазоне 12790-12791 на странице товаров, и статус код 200 OK.
