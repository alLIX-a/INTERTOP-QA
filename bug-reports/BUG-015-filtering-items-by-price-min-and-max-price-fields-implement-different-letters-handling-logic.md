# BUG-015: Фильтрация товаров по цене — поля минимальной и максимальной цены реализуют разную логику обработки буквенного значения

| Поле | Значение |
|---|---|
| **ID** | BUG-015 |
| **Серьезность** | Средняя |
| **Приоритет** | Низкий |

## Предусловия

1. Страница товаров открыта
2. Пользователь может быть авторизован или нет

## Тестовые данные

1. Буквенное значение: b

## Шаги воспроизведения

1. Нажать на кнопку "Цена"
2. В выпадающем меню вставить буквенное значение в правое поле меню
3. Нажать на кнопку "Ок"

## Ожидаемый результат

Система корректирует значение верхней границы до максимально допустимого. Применяется и отображается под списком фильтров фильтр с откорректированной верхней границей и нижней границей равной минимально допустимой. Показывается список товаров, чья стоимость находится в пределах заданного диапазона.

Логика обработки идентична логике обработки буквенного значения в поле минимальной цены (нижняя граница корректируется до минимально допустимой, применяется фильтр с откорректированной нижней границей).

## Фактический результат

Система не откорректировала значение верхней границы и не применила фильтр. Сообщение об ошибке показано не было.
